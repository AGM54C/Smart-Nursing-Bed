## 作品创意（≤500字）

针对我国老龄化加剧、慢性病患者长期护理需求激增，而传统人工巡检存在监测盲区大、夜间响应滞后、护理人力不足等痛点，本作品打造"端-边-云"三层融合的智能护理病床 AI 解决方案。端层基于 ESP32-S3 采集心率、血氧、体温、血压、六轴姿态与 8×8 压力矩阵等六类体征，采用 **TuyaOpen 开发框架**接入云端；边层在树莓派部署自研轻量 PostureMobileNet 模型（ONNX Runtime，<2ms）实现睡姿分类与三级受压预警，并构建"感知→决策→执行"AI 自主决策闭环（低血氧自动抬背、受压自动翻身、跌倒告警）；云层依托**涂鸦 AI 云开发者平台**实现设备接入、双链路数据上报与命令下发，结合**涂鸦智能体开发平台与大模型**完成健康报告、专科护理咨询与家属关注点分析，并提供全流程语音 AI 陪护。系统创新引入**星闪 NearLink（Hi3863）**室内高精度定位，配合循迹、RFID 与超声避障实现病床自主寻房导航；面向医护与家属的**涂鸦 App 小程序 / 微信小程序**及 Web 数字孪生 3D 界面实现多角色远程协同看护。基于 Dempster-Shafer 证据理论融合视觉、压力、体征三路证据，显著降低误报，为慢病患者提供全天候、智能化、可远程的连续护理。

## 技术架构

| 层次 | 内容 |
|---|---|
| **感知层** | ESP32-S3-N16R8 主控；MAX30102 心率血氧 + MLX90614 红外测温 + MPU6050 六轴姿态（I2C 共线）；8×8 Velostat 压力矩阵（双 CD74HC4051 多路复用行列扫描）；HK-2000B+ 脉搏波估算血压；CSI V2 摄像头 MJPEG 推流；**星闪 NearLink Hi3863 SLE 测距标签/锚点**（室内定位） |
| **传输层** | WiFi + MQTT（本地 Mosquitto，6 个 Topic）；**涂鸦 AI 云平台设备接入双链路**：MQTTS（TLS 加密、设备三元组认证、标准命令下发 Topic）；MQTT Bridge 经 HTTP 转发至云端；MJPEG 局域网推流；星闪 SLE 测距数据回传 |
| **控制层** | 树莓派 4B 边缘 AI 推理（PostureCNN 15K / PostureMobileNet 45K，引擎优先级 ONNX Runtime > MobileNet > CNN > 规则）；AI 自主决策 8 条优先级规则引擎；自主导航（5 路 TCRT5000 循迹 + RC522 RFID 病房识别 + 3 路 HC-SR04 超声避障 + L298N 驱动 4WD + PID 调速 + 星闪定位/卡尔曼滤波）；12V 电动推杆靠背升降 |
| **软件及开发环境** | 云端 Express 4.18 + SQLite3（WAL，7+2 表）+ JWT/RBAC 四角色；前端原生 HTML/CSS/JS + Chart.js + **Three.js 数字孪生 3D**；**涂鸦 App 小程序 / 微信小程序（涂鸦智能 IoT 小程序 SDK）**；端侧 **TuyaOpen（C 语言，ESP32-S3）**；边缘 Python（paho-mqtt、PyTorch、Flask 遥控）；星闪固件 C 语言（Hi3863）；PM2 进程管理 + Nginx 反向代理 |
| **云应用** | **涂鸦 AI 云开发者平台**设备接入与命令下发；**涂鸦智能体开发平台 + 涂鸦大模型**（健康报告 / 专科护理咨询 / 压力矩阵深度分析 / 家属关注点提取）；SiliconFlow TeleSpeechASR + CosyVoice2 语音识别与合成；区块链存证 |
| **其他内容** | 5 种专科护理 AI Agent（高位截瘫/糖尿病/中风/慢阻肺/通用，独立阈值与提示词，基于涂鸦智能体平台编排）；基于 Dempster-Shafer 证据理论的多模态融合告警（视觉跌倒 + 压力 + 体征三路证据 → 四级告警）；自然语言控制（大模型意图解析 → 11 种动作）；语音情感识别 SER；JWT+RBAC 多角色协作；联邦学习；区块链存证；作业成本与工业设计见设计文档 |
